Этот скрипт создан для генерации конфига bacula, адаптированного к конфигурации применяемой в Мариинском театре.

Принцип его работы следующий. Скрипт make_bacula_conf.py генерирует в стандартный выход конфиг для bacula опираясь
на шаблоны (расположенные в папочке template). Данный скрипт работает опираясь на конфиг script.conf. По сути это
единственный аргумент который принимает этот скрипт.
Таким образом станадртный процесс добавления нового клиента сотстоит из следующих пунктов:
- Добавить имя, пароль, и необходимые наборы файлов в файл script.conf
- Запустить: make_bacula_conf.py -c script.conf
- Проверить все ли верно, и если да то перенаправить вывод в какой-нибудь файл

Скрипт create_script_conf.py как видно из его имени служит для создания файла script.conf. Информацию он может брать
из двух источников: 1) из папочки credentials (она генерируется ansible), или уже существующего конфига bacula. На
выходе мы получим только имена клиентов, адреса и пароли, вручную нужно будет проставить желаемые наборы. По идее
если указать скрипту на уже существующий конфиг, он постарается найти и обновить значения для того или иного клиента,
ну а если не найдет, создаст.

В случае если нужно что нибудь изменить в параметрах бакулы, вам нужно лезть в папочку template. Язык шаблонов - mako,
не самый сложный язык, разобраться можно. Из скрипта происходит обращение к единственному шаблону base.mako и уже в нем
связываются все остальные шаблоны и шаблончики.

Файлы и их краткое описание:
─ create_script_conf.py		- скрипт для генерации/обновления script.conf
─ credentials 			- папка с парами имя/пароль. Необходим для работы create_script_conf.py
─ make_bacula_conf.py		- скрипт для генерации bacula.conf. Для его работы нужен script.conf
─ readme.txt			- ума не приложу, что это за файл
─ script.conf			- самое главное в этой папке.
─ template			- набор шаблонов необходимых для работы make_bacula_conf.py
